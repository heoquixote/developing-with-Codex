# 1-4. 모델 선택과 실행 모드 이해하기

Codex를 오래 쓰다 보면 좋은 결과를 만드는 핵심이 프롬프트만이 아니라는 점을 알게 됩니다. 어떤 모델을 쓰는지, 로컬에서 돌릴지 클라우드로 위임할지, 빠른 왕복을 우선할지 깊은 추론을 우선할지가 결과 품질과 작업 비용을 크게 좌우합니다.

- gpt-5.4와 gpt-5.4-mini를 언제 쓰는가
- 더 무거운 작업에서 Codex 계열 모델을 고르는 기준

- 비용·속도·정확도의 균형
- 정리

## gpt-5.4와 gpt-5.4-mini를 언제 쓰는가

가볍게 이해하면 됩니다. `gpt-5.4-mini`는 빠른 왕복과 저비용 실험에 유리하고, `gpt-5.4`는 더 복잡한 맥락 유지와 깊은 추론이 필요한 작업에 유리합니다.

실무에서는 다음처럼 나누면 편합니다.

- 빠른 코드 탐색, 문서 다듬기, 작은 수정은 `gpt-5.4-mini`
- 설계 비교, 다단계 디버깅, 복잡한 리팩터링 검토는 `gpt-5.4`

중요한 점은 "큰 모델이 항상 낫다"가 아니라 작업의 실패 비용에 맞는 모델을 고르는 것입니다.

## 더 무거운 작업에서 Codex 계열 모델을 고르는 기준

무거운 작업일수록 모델 선택 기준은 세 가지로 정리됩니다.

- 여러 파일과 긴 문맥을 함께 다뤄야 하는가
- 중간 판단을 여러 번 이어가야 하는가
- 잘못 바꾸면 되돌리기 비용이 큰가

이 세 조건이 강할수록 더 깊은 추론과 안정적인 문맥 유지가 중요해집니다. 반대로 이미 구조가 명확하고 사람이 거의 정답을 알고 있는 작업이라면 가벼운 모델로도 충분할 수 있습니다.

## Local 작업과 Cloud 작업의 차이

로컬 작업은 현재 작업 중인 저장소와 바로 연결되어 있다는 점이 가장 큰 장점입니다. 파일을 읽고, 테스트를 돌리고, 실제 수정 결과를 바로 확인하는 데 적합합니다. 반면 클라우드 작업은 시간이 오래 걸리거나 병렬 가치가 높은 작업을 분리해서 돌릴 때 유리합니다.

차이를 실무 언어로 바꾸면 이렇습니다.

- 로컬은 "지금 이 코드베이스를 안전하게 다루는 손"
- 클라우드는 "시간이 걸리는 작업을 분산 처리하는 별도 실행 슬롯"

따라서 현재 브랜치에서 세밀하게 수정할 때는 로컬이 자연스럽고, 여러 아이디어를 병렬 비교하거나 오래 걸리는 탐색을 맡길 때는 클라우드가 더 적합합니다.

## 비용·속도·정확도의 균형

이 세 요소는 동시에 최대로 가져가기 어렵습니다. 일반적으로 속도를 높이면 깊이가 줄고, 정확도를 높이려면 더 많은 추론과 검증이 필요해 비용이 올라갑니다.

실무에서 좋은 균형점은 다음과 같습니다.

- 탐색 단계에서는 속도를 우선합니다.
- 설계 결정 단계에서는 정확도를 우선합니다.
- 구현 단계에서는 변경 범위를 줄여 비용을 통제합니다.
- 검증 단계에서는 모델보다 테스트와 diff를 더 믿습니다.

즉, 비용을 아끼는 가장 좋은 방법은 항상 작은 모델을 쓰는 것이 아니라, 큰 작업을 작은 검증 가능한 단위로 쪼개는 것입니다.

## 정리

모델 선택은 취향의 문제가 아니라 작업 설계의 일부입니다. 어떤 모델이 더 똑똑한가보다, 어떤 단계에서 어떤 수준의 추론이 필요한가를 먼저 판단해야 합니다. Codex를 잘 쓰는 사람은 프롬프트만 잘 쓰는 사람이 아니라, 모델과 실행 모드를 작업 구조에 맞게 배치하는 사람입니다.

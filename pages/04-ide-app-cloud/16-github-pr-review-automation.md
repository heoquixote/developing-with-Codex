정리

PR 리뷰 자동화는 Codex 활용에서 가장 오해가 많은 영역 중 하나입니다. 많은 팀이 "사람 리뷰를 대체할 수 있는가"를 먼저 묻지만, 더 중요한 질문은 "사람 리뷰어가 더 중요한 문제에 집중하게 만들 수 있는가"입니다.


## 1. @codex review 사용법

`@codex review`의 핵심 가치는 빠른 1차 스크리닝입니다. 변경 범위를 읽고, 버그 가능성이나 테스트 누락, 설계상 부작용을 먼저 짚어줌으로써 사람이 리뷰해야 할 부분을 압축해줍니다.

특히 다음 상황에서 효과적입니다.

- 작은 PR이 많이 쌓일 때
- 리뷰어가 코드베이스 전체를 다 알지 못할 때
- 회귀 위험과 테스트 누락을 빠르게 점검하고 싶을 때


## 2. 자동 리뷰 설정하기

자동 리뷰를 도입할 때 중요한 것은 모든 PR에 같은 강도로 적용하지 않는 것입니다. 문서 수정 PR과 인증 흐름 변경 PR의 리뷰 기대 수준은 다릅니다.

좋은 도입 방식은 다음과 같습니다.

- 먼저 비차단형 코멘트로 시작합니다.
- 팀이 유용성을 확인한 뒤 범위를 넓힙니다.
- 고위험 영역은 사람 승인 중심으로 유지합니다.

자동화는 도입보다 운영이 더 중요합니다. 노이즈가 많으면 리뷰어는 금방 무시하기 시작합니다.


## 3. 리뷰 가이드라인 커스터마이즈

리뷰 품질을 높이려면 팀의 기준을 분명히 해야 합니다. 예를 들어 어떤 팀은 성능 저하를 가장 민감하게 보고, 어떤 팀은 API 호환성과 보안 이슈를 더 중요하게 볼 수 있습니다.

가이드라인에는 보통 다음 요소가 들어갑니다.

- 리뷰 우선순위
- 고위험 변경 유형
- 코멘트 톤과 형식
- 테스트 기대 수준

기준이 명확할수록 자동 리뷰는 더 유용해지고, 사람이 읽을 때도 맥락이 맞아집니다.


## 4. 사람 리뷰어와 Codex 리뷰의 역할 분리

Codex 리뷰는 반복적이고 구조적인 체크에 강합니다. 반면 사람 리뷰어는 제품 맥락, 사용자 영향, 팀 기준, 장기 설계 방향을 더 잘 봅니다. 둘을 같은 역할로 놓으면 실망하기 쉽고, 다른 역할로 놓으면 시너지가 큽니다.

실무적으로는 다음처럼 나누는 편이 좋습니다.

- Codex: 버그 가능성, 누락된 테스트, 명백한 회귀 위험
- 사람: 우선순위 판단, 제품 설계, 조직 맥락, 예외 승인


## 정리

PR 리뷰 자동화의 목적은 사람을 빼는 것이 아니라, 사람이 더 중요한 판단을 하게 만드는 것입니다. 좋은 자동 리뷰는 리뷰를 대신하는 도구가 아니라, 리뷰의 초점을 선명하게 만드는 도구입니다.

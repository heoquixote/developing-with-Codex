# 4. IDE·App·Cloud로 확장하기

## 목차

1. 이 파트가 끝나면 할 수 있는 일

이 파트는 로컬 실습에서 병렬 작업과 팀 협업으로 넘어가는 구간입니다. 여기서부터 Codex는 개인 도구가 아니라 팀 생산성 체계의 일부가 됩니다.

로컬 CLI가 한 사람의 작업 루프를 강화하는 도구라면, IDE 확장, App, Cloud, GitHub 리뷰 자동화는 여러 맥락과 여러 작업 단위를 동시에 굴리기 위한 계층입니다.


## 1. 이 파트가 끝나면 할 수 있는 일

- IDE 안에서 Codex를 페어 프로그래머처럼 활용할 수 있습니다.
- worktree와 스레드를 분리해 병렬 작업을 설계할 수 있습니다.
- 시간이 오래 걸리는 작업을 Cloud로 위임할 수 있습니다.
- GitHub PR 리뷰 자동화를 사람 리뷰와 분리해 운용할 수 있습니다.

이 파트의 핵심은 기능 이해가 아니라 역할 분담입니다. 어떤 작업을 어디에서 돌릴 때 가장 효율적인지 감각을 만드는 것이 중요합니다.

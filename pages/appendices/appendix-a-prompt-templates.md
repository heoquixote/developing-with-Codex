이 Appendix는 자주 쓰는 요청 패턴을 빠르게 재사용할 수 있도록 템플릿 형태로 정리한 문서입니다.

## 목차

- 1. 코드베이스 탐색 템플릿
- 2. 기능 추가 템플릿
- 3. 버그 수정 템플릿
- 4. 리팩터링 템플릿
- 5. 문서화 템플릿
- 6. PR 리뷰 템플릿
- 7. 데이터/AI 엔지니어링 작업 템플릿

## 1. 코드베이스 탐색 템플릿

```text
이 저장소의 {기능/모듈} 흐름을 파악하고 싶습니다.
다음 순서로 진행해주세요.
1. 관련 파일과 진입점을 찾기
2. 요청 흐름 또는 호출 관계 설명
3. 핵심 데이터 구조 정리
4. 수정 시 주의할 위험 지점 정리
출력은 "개요 / 핵심 파일 / 실행 흐름 / 주의점" 형식으로 주세요.
```

## 2. 기능 추가 템플릿

```text
다음 요구사항을 최소 구현 기준으로 반영해주세요.

목표:
{무엇을 추가할지}

범위:
{수정 가능한 파일/모듈}

제약:
- 기존 API 호환성 유지
- 테스트 추가
- 불필요한 리팩터링 금지

작업 순서:
1. 구현 계획 제시
2. 코드 수정
3. 테스트 추가 또는 수정
4. 변경 요약과 검증 결과 보고
```

## 3. 버그 수정 템플릿

```text
다음 버그를 high-confidence change로 수정해주세요.

증상:
{오류 메시지 또는 관찰된 현상}

재현 단계:
1. ...
2. ...
3. ...

원하는 결과:
{정상 동작}

주의사항:
- 재현 가능한 원인부터 확인
- 추측성 변경 금지
- 회귀 테스트가 가능하면 추가
```

## 4. 리팩터링 템플릿

```text
이 코드를 동작 변경 없이 리팩터링해주세요.

목표:
- 함수 분리
- 중복 제거
- 읽기 쉬운 이름 정리

제약:
- 공개 인터페이스 유지
- 테스트가 깨지면 안 됨
- 한 번에 큰 구조 변경 금지

출력:
- 리팩터링 계획
- 변경된 코드
- 동작 보존 근거
```

## 5. 문서화 템플릿

```text
현재 구현을 바탕으로 문서를 업데이트해주세요.

필수 포함:
- 기능 목적
- 사용 방법
- 입력/출력 예시
- 제약사항 또는 실패 케이스

문체:
- 실무 문서 스타일
- 불필요한 마케팅 표현 금지
```

## 6. PR 리뷰 템플릿

```text
이 변경을 코드 리뷰 관점에서 검토해주세요.

우선순위:
1. 버그 가능성
2. 회귀 위험
3. 테스트 누락
4. 설계상 부작용

출력 형식:
- Findings first
- 심각도 순 정렬
- 파일/라인 기준 설명
- 마지막에 남은 리스크 요약
```

## 7. 데이터/AI 엔지니어링 작업 템플릿

```text
다음 데이터 파이프라인 작업을 정리해주세요.

목표:
{파이프라인 개선 또는 자동화 목표}

현재 상태:
{Python/SQL/배치 구조}

원하는 결과:
- 재현 가능한 실행 방법
- 데이터 품질 체크 포함
- 실패 시 원인 파악 가능
- 문서화 포함

출력:
- 작업 계획
- 코드 수정
- 검증 방법
- 운영 체크포인트
```

프롬프트 템플릿의 핵심은 길이가 아니라 구조입니다. 목표, 범위, 제약, 출력 형식이 분명하면 Codex의 결과는 안정되고, 반대로 이 네 요소가 빠지면 결과가 흔들립니다.

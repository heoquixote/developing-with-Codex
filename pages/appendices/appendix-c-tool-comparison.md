이 비교표는 어떤 작업을 어디에서 실행하는 것이 자연스러운지 빠르게 판단하기 위한 요약 자료입니다.

| 구분 | CLI | IDE | App | Cloud |
| --- | --- | --- | --- | --- |
| 가장 잘 맞는 작업 | 코드베이스 탐색, 테스트, 명령 실행 | 파일 단위 수정, 빠른 왕복 | 스레드 분리, 병렬 작업 관리 | 오래 걸리는 작업 위임, 병렬 실험 |
| 강점 | 저장소 전체 맥락, 검증 용이 | 편집 흐름과 밀착 | 작업 단위 분리 | 시간 절약, 병렬성 |
| 약점 | 초보자에게 진입장벽 | 전체 구조 시야 부족 | 로컬만큼 세밀한 편집은 약할 수 있음 | 결과 수렴과 검토가 별도 필요 |
| 추천 상황 | 버그 재현, 테스트 확인, 리팩터링 검증 | UI 수정, 함수 정리, 짧은 질의응답 | 여러 과제 동시 진행 | 대규모 탐색, 대안 비교 |
| 주의점 | 범위 넓은 요청 주의 | 현재 파일에 과몰입 가능 | 작업 충돌 관리 필요 | 완성본으로 오해하지 말 것 |

## 빠른 선택 기준

- 지금 당장 테스트와 diff를 확인해야 하면 CLI
- 현재 열어둔 파일을 중심으로 빠르게 손보려면 IDE
- 작업을 여러 개로 나눠 관리해야 하면 App
- 오래 걸리는 조사나 병렬 초안이 필요하면 Cloud

중요한 것은 한 도구를 만능으로 쓰지 않는 것입니다. 좋은 협업은 실행 환경을 작업 특성에 맞게 바꾸는 데서 나옵니다.

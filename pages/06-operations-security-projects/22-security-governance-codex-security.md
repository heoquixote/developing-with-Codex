# 6-2. 보안, 거버넌스, 선택 주제: Codex Security

## 목차

1. 승인 경계와 권한 관리
2. 저장소 신뢰와 프로젝트 설정
3. 보안 검토 워크플로
4. 보안 검토 체크리스트
5. 자주 하는 실수
6. Codex Security를 어디까지 책에 넣을 것인가
정리

보안은 Codex 활용을 막는 주제가 아니라, 오래 쓰기 위해 반드시 설계해야 하는 주제입니다. 특히 코드 수정, 명령 실행, 외부 연결, 자동화가 결합되면 권한과 책임 경계가 매우 중요해집니다.


## 1. 승인 경계와 권한 관리

어떤 작업은 자동으로 실행해도 되지만, 어떤 작업은 반드시 사람이 승인해야 합니다. 이 기준은 편의가 아니라 영향 범위로 정해야 합니다.

- 읽기 전용 탐색은 비교적 자동화가 쉽습니다.
- 파일 수정은 범위와 위치가 중요합니다.
- 배포, 인프라, 비밀정보 접근은 가장 보수적으로 관리해야 합니다.

권한 관리는 사용자의 자유를 제한하는 것이 아니라, 실수를 국소화하는 장치입니다.

실무에서는 작업 유형별로 승인 강도를 다르게 가져가는 편이 현실적입니다.

| 작업 유형 | 권장 승인 수준 | 이유 |
| --- | --- | --- |
| 코드 읽기, 탐색 | 낮음 | 영향이 읽기 범위에 머무릅니다. |
| 문서 수정 | 중간 | 영향 범위는 작지만 잘못된 정보가 남을 수 있습니다. |
| 애플리케이션 코드 수정 | 중간~높음 | 회귀 위험과 테스트 범위 확인이 필요합니다. |
| 운영 설정 변경, 배포, 비밀정보 접근 | 매우 높음 | 실패 비용과 보안 영향이 큽니다. |


## 2. 저장소 신뢰와 프로젝트 설정

모든 저장소를 같은 신뢰 수준으로 다룰 수는 없습니다. 실험용 개인 저장소와 운영 서비스 저장소는 위험도와 통제 방식이 다릅니다. 따라서 프로젝트 설정은 저장소 특성에 맞게 달라져야 합니다.

실무에서는 보통 다음 요소를 함께 봅니다.

- 민감정보 포함 여부
- 배포와 직접 연결되는지
- 협업 인원이 얼마나 많은지
- 변경 실패 비용이 얼마나 큰지

같은 모델을 쓰더라도 저장소가 바뀌면 정책도 바뀌어야 합니다. 개인 실험 저장소에서는 빠른 반복이 중요할 수 있지만, 운영 서비스 저장소에서는 승인 경계와 감사 가능성이 훨씬 중요합니다.


## 3. 보안 검토 워크플로

보안 검토는 별도의 팀만 하는 특별 절차가 아니라, 변경 흐름 안에 녹아 있어야 합니다. 예를 들어 인증, 권한, 데이터 접근, 외부 연동, 로그 마스킹 같은 영역은 평소 리뷰 기준 자체가 달라야 합니다.

Codex도 여기에 기여할 수 있습니다. 잠재적 위험 포인트를 먼저 스크리닝하고, 정책 위반 가능성을 체크하고, 보안 리뷰 체크리스트 초안을 만드는 데 유용합니다. 다만 최종 판단은 여전히 사람 책임입니다.


## 4. 보안 검토 체크리스트

- 변경이 인증, 권한, 데이터 접근 경계를 건드리는지 확인했습니다.
- 로그와 에러 메시지에 민감정보가 섞이지 않는지 점검했습니다.
- 외부 연동 추가 시 토큰과 권한 범위를 최소화했습니다.
- 자동 실행보다 사람 승인 지점이 더 적절한 작업인지 검토했습니다.
- 저장소 신뢰 수준에 맞는 프로젝트 설정을 적용했습니다.


## 5. 자주 하는 실수

- 모든 저장소에 같은 승인 정책을 적용합니다.
- 코드 수정 위험은 보면서 문서나 설정 변경 위험은 과소평가합니다.
- 보안 검토를 마지막 단계의 별도 이벤트로만 생각합니다.
- Codex가 위험을 찾을 수 있다는 이유로 최종 사람 책임을 흐립니다.


## 6. Codex Security를 어디까지 책에 넣을 것인가

보안 주제를 책에 다룰 때는 두 층을 나누는 것이 좋습니다.

- 모든 독자가 알아야 하는 기본 원칙
- 조직 성숙도에 따라 깊이가 달라지는 선택 주제

이 구분이 없으면 입문자는 과도하게 어려워지고, 실무자는 너무 얕다고 느끼기 쉽습니다. 따라서 이 책에서는 승인 경계, 권한, 저장소 신뢰, 리뷰 흐름을 기본 범위로 두고, 더 깊은 보안 기능은 선택 주제로 다루는 구성이 자연스럽습니다.


## 정리

보안과 거버넌스는 Codex 활용의 반대편에 있는 개념이 아니라, Codex 활용을 실무에 맞게 만드는 조건입니다. 신뢰는 자유에서 나오지 않고, 적절한 경계에서 나옵니다.
